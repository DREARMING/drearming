二维码的生成和识别都是采用了二维码第三方类库 - Zxing

二维码的生成

调用Zxing第三方库可以很容易生成二维码，我们用到的类是 MultiFormatWriter

<% highlight java %>

public final class MultiFormatWriter implements Writer {
@Override public BitMatrix encode(String contents,
BarcodeFormat format, int width, int height) throws WriterException { return encode(contents, format, width, height, null); }
@Override
public BitMatrix encode(String contents, BarcodeFormat format, int width, int height, Map<EncodeHintType,?> hints) throws WriterException { Writer writer; switch (format) { ... case QR_CODE: writer = new QRCodeWriter(); break; ... default: throw new IllegalArgumentException("No encoder available for format " + format); } return writer.encode(contents, format, width, height, hints); } }

<% endhighlight %>

可以很容易看出，只需要我们这样子使用就可以得到一张二维码矩阵

<% highlight java %>

/* 生成一个由点构成的矩阵，矩阵的点分布情况，代表着写进去的内容，可由对应Reader读出 @param content 是要写进二维码图片的内容，我们扫描的目的就是获取这内容 @param BarCodeFormat.QR_CODE 这个从上面就可以看出，这个是配置哪种Writer，我们指定的是二维码，当然也有一维码这种 @param width 指定生成二维码矩阵的宽 @param height 指定生成二维码矩阵的高 @params hints 是一个map类型，告诉要写contents的writer，如何写这个content（utf-8编码，容错度的大小，等等情况） **/ BitMatrix matrix = new MultiFormatWriter().encode(contents,BarcodeFormat.QR_CODE, width, height,hints);

<% endhighlight %>

我们得到一个二维码矩阵后，再将它转化成一张二维码图片

<% highlight java %>

private static Bitmap matrixToImage(BitMatrix matrix){ Bitmap image = Bitmap.createBitmap(matrix.getWidth(),matrix.getHeight(),Bitmap.Config.RGB_565); for(int i = 0 ; i<matrix.getWidth();i++){ for(int j = 0; j < matrix.getHeight();j++){ //假如点矩阵中的这个点存在，就在image中描黑色，当然颜色可以随你定义。 //循环结束，我们就完美地在image中描绘出点矩阵 image.setPixel(i, j, (matrix.get(i, j)?BLACK:WHITE)); } } return image; }

<% endhighlight %>

到这里，你就算是完成了二维码图片的生成了，但是有时候我们需要加入logo；

<% highlight java %>

/**
* 设置 logo * @param matrixImage 源二维码图片 * @return 返回带有logo的二维码图片 * @throws IOException * @author Administrator mvp */ public static Bitmap logoMatrix(Bitmap matrixImage,Bitmap logo){ /** * 用刚刚生成的二维码图片，构建画布对象 */ Canvas canvas = new Canvas(matrixImage); //获取二维码图片的宽高，为了正确计算logo所在的位置 int matrixWidth = matrixImage.getWidth(); int matrixHeigh = matrixImage.getHeight(); Paint paint = new Paint(); paint.setStyle(Paint.Style.FILL); //logo的位置，在二维码图片的 2/5 - 3/5间 Rect logoRect = new Rect(matrixWidth/5*2,matrixHeigh/5*2, matrixWidth/5*3, matrixHeigh/5*3); paint.setColor(WHITE); //添加一個比logo要大的白色背景 //将白色背景生成 RectF rectf = new RectF(logoRect.left-3,logoRect.top-3,logoRect.right+3,logoRect.bottom+3); //第二个参数和第三个参数指的是圆角的大小半径 canvas.drawRoundRect(rectf, 5, 5, paint); //将logo画上 canvas.drawBitmap(logo, null, logoRect, paint); //给logo外围加一条边线 paint.setStyle(Paint.Style.STROKE); paint.setStrokeWidth(1); paint.setColor(Color.GRAY); canvas.drawRoundRect(new RectF(logoRect), 5, 5, paint); return matrixImage ; }

<% endhighlight %>

二维码的扫描

二维码的扫描原理非常复杂，有兴趣地再往下面看看，现在简单介绍一下项目中的应用

背景：我们项目中有个现成的zxing库，里面有一个现成的扫描界面 – MipcaActivityCapture ，我们只需要调用这个Activity，就可以打开二维码扫描界面，然后扫到二维码后会自动把读到的信息返回到上一个Activity的onActivityResult方法中

先在manifest文件中注册

<% highlight java %>

<activity android:name="com.xxx.xxx.zxing.MipcaActivityCapture" android:configChanges="orientation|keyboardHidden" android:screenOrientation="portrait" android:windowSoftInputMode="stateAlwaysHidden">

<% endhighlight %>

我们定义了一个button，设置监听事件,用于打开扫描界面

<% highlight java %>

class ScanSQCodeListener implements OnClickListener{ @Override public void onClick(View v) { // TODO Auto-generated method stub Intent intent = new Intent(); intent.setClass(mContext, MipcaActivityCapture.class); startActivityForResult(intent, SCANNIN_GREQUEST_CODE); }}

<% endhighlight %>

###扫描界面

当我们扫到二维码后，在Activity中监听返回结果

<% highlight java %>

@Override protected void onActivityResult(int requestCode, int resultCode, Intent data) { super.onActivityResult(requestCode, resultCode, data); switch (requestCode) { case SCANNIN_GREQUEST_CODE: //程序内部设定的结果码就是RESULT_OK if(resultCode == RESULT_OK){ Bundle bundle = data.getExtras(); //显示扫描到的内容 NNotify.showToast(getApplicationContext(), bundle.getString("result")); String resultString = bundle.getString("result"); //考虑查询数据库，确认userid和cardId是否存在 refresh("QRCode_found",resultString); } break; } }

<% endhighlight %>

到此，二维码扫描和生成就集成成功了

##扫描界面的流程原理分析

背景：如果不懂Camera的工作方式，请查看官网 API Guide

  1. 我们先定义个surfaceView用来显示Camera拍到的图像
  2. 界面中那个ViewfinderView，是一个自定义控件，不断刷新，形成一个扫描动画，同时里面有一个hashset用来存储可能的结果点，在界面刷新时显示不同的颜色，给人一种好像扫描到东西的感觉。(备注：这些结果点是真的存在，是Camera扫到的可能结果，然后动态添加到这个自定义控件中，随着刷新显示出来)

###我们来讨论一下是Camera是如何识别到二维码的整个过程

首先我们要清楚Camera的PreviewCallback的工作原理，这个需要sdk等级大于3

<% highlight java %>

//当调用camera.setOneShotPreviewCallback()，那么每当surfaceView 的 preview发生变化时，都会回调PreviewCallBack的 //public void onPreviewFrame(byte[] data, Camera camera)方法； public void requestPreviewFrame(Handler handler, int message) { if (camera != null && previewing) { previewCallback.setHandler(handler, message); if (useOneShotPreviewCallback) { camera.setOneShotPreviewCallback(previewCallback); } else { camera.setPreviewCallback(previewCallback); } } } //PreviewCallback 代码 //data是当前拍到的照片数据，会不断返回给传递给PreviewCallBack的handler，这里是DecodeHandler解码线程的handler； public void onPreviewFrame(byte[] data, Camera camera) { Point cameraResolution = configManager.getCameraResolution(); if (!useOneShotPreviewCallback) { camera.setPreviewCallback(null); } if (previewHandler != null) { Message message = previewHandler.obtainMessage(previewMessage, cameraResolution.x, cameraResolution.y, data); message.sendToTarget(); //这里至为null是因为，为了避免不断向解码线程的handler发送图片数据， //这是不正确的行为，正确的行为是：解析完拍到的图片后，如果解析失败后，才有必要重新设绑定handler，再接收新拍到的图片； previewHandler = null; } else { Log.d(TAG, "Got preview callback, but no handler for it"); } }

<% endhighlight %>

从上面的注释我们就明白了，当我们摄像机再移动的过程中，不断在拍照中，会调用PreviewCallback的onPreviewFrame()方法，将拍到图像以byte[]的形式传递进去，这个时候绑定在解码线程的的handler，就会发送信息到解码线程中去；

为什么是解码线程的Handler？来，一步一步追着看

首先MipActivity 在初始化Camera的时候就初始化了CaptureActivityHandler

<% highlight java %>

private void initCamera(SurfaceHolder surfaceHolder) { try { CameraManager.get().openDriver(surfaceHolder); } catch (IOException ioe) { return; } catch (RuntimeException e) { return; } if (handler == null) { //注意第一个参数，把MipActivity自身传了进去，这个为了方便传递给DecodeHandler handler = new CaptureActivityHandler(this, decodeFormats, characterSet); } } //CaptureActivityHandler.java //初始化CaptureActivityHandler的时候，我们开启了DecodeThread，并且把MipActivity的引用传递进去，又调用了 //RestartPreviewAndDecode() public CaptureActivityHandler(MipcaActivityCapture activity, Vector<BarcodeFormat> decodeFormats, String characterSet) { this.activity = activity; decodeThread = new DecodeThread(activity, decodeFormats, characterSet, new ViewfinderResultPointCallback(activity.getViewfinderView())); decodeThread.start(); state = State.SUCCESS; // Start ourselves capturing previews and decoding. CameraManager.get().startPreview(); restartPreviewAndDecode(); } //哈哈，看到这里了没有，它调用了PreviewCallback的requestPreviewFrame的方法，把decodeThread的handler传递给 //PreviewCallback，这样子，上面说的原理就通了 private void restartPreviewAndDecode() { if (state == State.SUCCESS) { state = State.PREVIEW; CameraManager.get().requestPreviewFrame(decodeThread.getHandler(), R.id.decode); CameraManager.get().requestAutoFocus(this, R.id.auto_focus); activity.drawViewfinder(); } }

<% endhighlight %>

我们看看解码线程如何处理二维码图片

<% highlight java %>

//解码线程的handler收到Message时会调用Decode方法，并把ViewFinderView的长和宽作为arg1，arg2传递了进去(上面翻翻就知道我说的有没有错了) @Override public void handleMessage(Message message) { switch (message.what) { case R.id.decode: //Log.d(TAG, "Got decode message"); decode((byte[]) message.obj, message.arg1, message.arg2); break; case R.id.quit: Looper.myLooper().quit(); break; } }
/** * Decode the data within the viewfinder rectangle, and time how long it took. For efficiency, * reuse the same reader objects from one decode to the next. * * @param data The YUV preview frame. * @param width The width of the preview frame. * @param height The height of the preview frame. */ private void decode(byte[] data, int width, int height) {
long start = System.currentTimeMillis(); Result rawResult = null; //modify here byte[] rotatedData = new byte[data.length]; for (int y = 0; y < height; y++) { for (int x = 0; x < width; x++) rotatedData[x * height + height - y - 1] = data[x + y * width]; } int tmp = width; // Here we are swapping, that's the difference to #11 width = height; height = tmp; PlanarYUVLuminanceSource source = CameraManager.get().buildLuminanceSource(rotatedData, width, height); BinaryBitmap bitmap = new BinaryBitmap(new HybridBinarizer(source)); try { rawResult = multiFormatReader.decodeWithState(bitmap); } catch (ReaderException re) { // continue } finally { multiFormatReader.reset(); } if (rawResult != null) { long end = System.currentTimeMillis(); Log.d(TAG, "Found barcode (" + (end - start) + " ms):\n" + rawResult.toString()); Message message = Message.obtain(activity.getHandler(), R.id.decode_succeeded, rawResult); Bundle bundle = new Bundle(); bundle.putParcelable(DecodeThread.BARCODE_BITMAP, source.renderCroppedGreyscaleBitmap()); message.setData(bundle); //Log.d(TAG, "Sending decode succeeded message..."); message.sendToTarget(); } else { Message message = Message.obtain(activity.getHandler(), R.id.decode_failed); message.sendToTarget(); } }

<% endhighlight %>

我也不清楚具体怎么处理图片的，这涉及到里面的内部原理，不是本文的重点，我们看到，当成功的时候会通过MipAcitivity获取到CaptureAcitivity的handler，在把解析到的数据传回去；如果失败的话，我们就会重新调用RestartPreviewAndDecode()方法，再把DecodeHandler的线程绑定到PreviewCallback中，继续接受拍到的新图片；

至此，整个二维码扫描界面的原理解析完成
