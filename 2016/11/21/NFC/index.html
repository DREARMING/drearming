NFC刷卡原理和功能实现

参考文献：Android API Guide的NFC

如果想直接了解系统中的NFC功能模块，直接跳到文章的非NDEF TAG部分进行阅读，因为公司用的饭卡都是属于MifareClassic技术的，不含有NDEF数据

如何检测手机是否支持NFC功能，或者是否打开了NFC功能

<% highlight java %>

<uses-feature android:name="android.hardware.nfc" android:required="true" />

<% endhighlight %>

筛选掉不支持NFC功能的手机

或者在应用中进行检测

<% highlight java %>

NFCAdapter adapter = NFCAdapter.getDefaultAdapter(); adapter==null 设备不支持NFC功能 adapter.isEnable() == false; 设备没有打开NFC功能

<% endhighlight %>

NFC是短距离传输技术，当我们支持NFC技术的手机触碰到NFC tag时，就会触发一个意图，试图找到对应的Activity，正因为是超短距离传输，我们应该默认地直接找到应用，而非存在多个Activity都能处理此Tag触发的意图，因为Activity Chooser被触发，我们要进行选择的话，很容易让我们的手机离开NFC TAG，从而导致连接失败的情况

如何定义一个Activity的Intent-filter，可以瞄准某类型的NFC TAG？

当支持NFC的Android手机检测到tag时，Intent-filter检查Action的顺序如下： 首先是 NEFT_DISCOVER -> TECH_DISCOVER ->TAG_DISCOVER

NDEF tag

首先一个NDEF tag的 NDEF Message 它有很多NDEF records，其中正确的形式封装，第一条NDEF Record里面应该有四个域存在：

    1. 一个 TNF 3bit 可以参照文档中的表格Table1，是指定tag的Type Name Format（类型名字格式，就是说你这个类型是什么，是一个uri，还是一个minetype，还是空的 等等），如果是well-know形式，会查看2中定义的类型（RTN）
    1. 一个 variable length type 参照Table2 ，定义tag确切的type类型（例如uri、text）
    1. 定义tag的id，不常用
    1. 负载在此条记录的数据（NDEF Message中有很多纪录，不要以为第一条纪录的数据就是全部）

<% highlight java %>

Activity中如何定义Intent-filter来筛选NDEF_DISCOVERD的tag，其MIME TYPE 类型是text/plain <intent-filter> <action android:name="android.nfc.action.NDEF_DISCOVERED"/> <category android:name="android.intent.category.DEFAULT"/> <data android:mimeType="text/plain" /> </intent-filter> //筛选的同样是NDEF_DISCOVERD，其类型是uri <intent-filter> <action android:name="android.nfc.action.NDEF_DISCOVERED"/> <category android:name="android.intent.category.DEFAULT"/> <data android:scheme="http" android:host="developer.android.com" android:pathPrefix="/index.html" /> /intent-filter>

<% endhighlight %>

创建一个URI TAG 记录 – TNF_ABSOLUTE_URI

<% highlight java %>

//参数如上面解释的： //1.是TNF，这里指定是URI类型 2.RTN 指定的URI，3.一个tag 的id，这里为空 4.数据 NdefRecord uriRecord = new NdefRecord( NdefRecord.TNF_ABSOLUTE_URI , "http://developer.android.com/index.html".getBytes(Charset.forName("US-ASCII")), new byte[0], new byte[0]); //上面的tag可以筛选的标签 <intent-filter> <action android:name="android.nfc.action.NDEF_DISCOVERED" /> <category android:name="android.intent.category.DEFAULT" /> <data android:scheme="http" android:host="developer.android.com" android:pathPrefix="/index.html" /> </intent-filter>

<% endhighlight %>

create a TNF_MIME_MEDIA NDEF record in the following ways:

<% highlight java %>

NdefRecord mimeRecord = NdefRecord.createMime("application/vnd.com.example.android.beam", "Beam me up, Android".getBytes(Charset.forName("US-ASCII"))); //也可以写成 NdefRecord mimeRecord = new NdefRecord( NdefRecord.TNF_MIME_MEDIA , "application/vnd.com.example.android.beam".getBytes(Charset.forName("US-ASCII")), new byte[0], "Beam me up, Android!".getBytes(Charset.forName("US-ASCII"))); <intent-filter> <action android:name="android.nfc.action.NDEF_DISCOVERED" /> <category android:name="android.intent.category.DEFAULT" /> <data android:mimeType="application/vnd.com.example.android.beam" /> </intent-filter>

<% endhighlight %>

###对于非NDEF数据的Tag，即类似饭卡那些不是标准的NDEF数据，如何筛选到**

if your activity filters for the ACTION_TECH_DISCOVERED intent, you must create an XML resource file that specifies the technologies that your activity supports within a tech-list set. Your activity is considered a match if a tech-list set is a subset of the technologies that are supported by the tag, which you can obtain by calling getTechList().

当tag发送的Intent的action是 ACTION_TECH_DISCOVERED ,必须定义一个tech-list的xml文档，当xml文档中有一个tech-list是tag所支持的技术的子集，就会被认为筛选到了，从而打开activity；我们可以通过getTechLIst()来获取tag中所支持的技术，获取到技术后，就能够通过这种技术的静态get(Tag tag)方法，对tag进行读写

<% highlight java %>

manifest文件中： <meta-data android:name="android.nfc.action.TECH_DISCOVERED" android:resource="@xml/nfc_tech_filter" /> nfc_tech_filter.xml文件 <resources xmlns:xliff="urn:oasis:names:tc:xliff:document:1.2" > <tech-list> <tech>android.nfc.tech.MifareClassic</tech> </tech-list> <tech-list> <tech>android.nfc.tech.MifareUltralight</tech> </tech-list> <tech-list> <tech>android.nfc.tech.NfcA</tech> </tech-list> </resources>

<% endhighlight %>

AAR – Android Application Records 指定唯一应用能接受此tag （4.0以后）

我们就是要用这个tag打开固定的应用，如果应用存在就打开，否则就跳到google中下载，不能打开非此应用的activity；如果此应用有activity匹配tag中的筛选条件，那么就打开应用的这个activity，否则打开应用的mainactivity；

AAR记录写入，最好不要是第一条NdefRecord，第一条是用来标识uri和mine type的，除非你的应用只筛选 AAR

<% highlight java %>

NdefMessage msg = new NdefMessage( new NdefRecord[] { ..., NdefRecord.createApplicationRecord("com.example.android.beam")}

<% endhighlight %>

如何读取NDEF Tag的内容？

<% highlight java %>

@Override protected void onNewIntent(Intent intent) { super.onNewIntent(intent); ... if (intent != null && NfcAdapter.ACTION_NDEF_DISCOVERED.equals(intent.getAction())) { Parcelable[] rawMessages = intent.getParcelableArrayExtra(NfcAdapter.EXTRA_NDEF_MESSAGES); if (rawMessages != null) { NdefMessage[] messages = new NdefMessage[rawMessages.length]; for (int i = 0; i < rawMessages.length; i++) { messages[i] = (NdefMessage) rawMessages[i]; } // Process the messages array. ... } } } <% endhighlight %>

非NDEF的TAG的操作

首先用getTechList()获取到支持的技术(字符串数组形式返回)，然后匹配一下是哪种技术，比如支持的是MifareClassic技术，那么它的读写操作如下：

<% highlight java %>

//订餐系统中当有卡临近手机时，会打开NfcReaderActivity，调用这个find()方法； public boolean find(Intent _intent) { // 得到是否检测到ACTION_TECH_DISCOVERED触发 String mAction = _intent.getAction(); if (NfcAdapter.ACTION_NDEF_DISCOVERED.equals(mAction)) { } else if (NfcAdapter.ACTION_TECH_DISCOVERED.equals(mAction)) { Tag tag = _intent.getParcelableExtra(NfcAdapter.EXTRA_TAG); Log.d("nfc", ""+NFunc.bytesToHexString(tag.getId())); if (tag != null) { String[] techList = tag.getTechList(); if (techList != null) { for (int i = 0; i < techList.length; i++) { String name = techList[i]; try { mCardHandles[fixTagIndex(name)] = fixCardTag(name, tag); } catch (Exception e) { } } } return true; } } else if (NfcAdapter.ACTION_TAG_DISCOVERED.equals(mAction)) { } return false; } private NCardHandle fixCardTag(String _name, Tag _tag) { if (_name != null) { _name = _name.substring(_name.lastIndexOf('.') + 1); if ("NfcA".equals(_name)) { return NNfcA.create(_tag); } else if ("NfcB".equals(_name)) { } else if ("NfcF".equals(_name)) { } else if ("NfcV".equals(_name)) { } else if ("IsoDep".equals(_name)) { return NIsoDep.create(_tag); } else if ("Ndef".equals(_name)) { } else if ("NdefFormatable".equals(_name)) { } else if ("MifareClassic".equals(_name)) { return NMifareClassic.create(_tag); } else if ("MifareUltralight".equals(_name)) { } } return null; }

<% endhighlight %>

务必注意：这个代码样例中，TAG里面封装的数据都是一开始定好的，比如这个块写什么，这个块能读出什么，哪个块是读密码，哪个块是写密码块，下面的例子只是单纯地展示一些读写的操作，而跟指定的块和扇区的值不存在必然关系

读操作

@Override public byte[] read(int _sector, int _block, byte[] _key) { MifareClassic mMifareClassic = MifareClassic.get(mTag); try { mMifareClassic.connect(); //读保护，只有给定的_key数组跟卡里面的读密码字节数组 匹配时，才能进行读卡操作； if (mMifareClassic.authenticateSectorWithKeyA(_sector, _key == null ? MifareClassic.KEY_DEFAULT : _key)) { return mMifareClassic.readBlock(mMifareClassic.sectorToBlock(_sector) + _block); } } catch (Exception e) { Log.e("", e.toString()); } finally { try { mMifareClassic.close(); } catch (Exception e2) { } } return null; }
